\Application\Entity

\Application\EntityUser

Entity representing a user of the application.

A user "has a" account owner, which is a Person, so User composes a Person. Inheritance is not the best option because there will be cases where an Interpreter and a User will both point to the same Person, and Doctrine doesn't have an inheritance strategy that works well with that.

$judges : \Doctrine\Common\Collections\ArrayCollection

Judge(s) to whom a user of hat Law Clerk or Courtroom Deputy is assigned.

Most of these users have one and only one judge, but there can be cases where they have more than one, hence the many-to-many rather than many-to-one. This is unidirectional.
